# 🏋️ Fitness Trainer Bot (v2 — несколько тренеров)

---

## 🚀 Быстрый старт

### Шаг 1 — Создайте бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather)
2. Напишите `/newbot`
3. Придумайте **название** (например: `Фитнес Студия`)
4. Придумайте **username** — только латиница, заканчивается на `bot` (например: `fitnes_studio_bot`)
5. Скопируйте токен вида: `1234567890:AAxxxxxxxxxxxxxxx`

### Шаг 2 — Настройте .env

```env
BOT_TOKEN=вставьте_сюда_токен
TRAINER_SECRET=придумайте_секретный_код
TIMEZONE=Europe/Moscow
```

`TRAINER_SECRET` — код, который вы сообщаете тренерам лично.
Только зная этот код, они смогут зарегистрироваться как тренеры.

**Часовые пояса:**
| Город       | TIMEZONE           |
|-------------|-------------------|
| Москва      | Europe/Moscow     |
| Киев        | Europe/Kiev       |
| Минск       | Europe/Minsk      |
| Алматы      | Asia/Almaty       |
| Ташкент     | Asia/Tashkent     |
| Новосибирск | Asia/Novosibirsk  |

### Шаг 3 — Установите зависимости

```bash
pip install -r requirements.txt
```

### Шаг 4 — Запустите

```bash
python main.py
```

## 📋 Как работает бот

### Для тренера
1. Написать `/start` → выбрать «Я тренер» → ввести секретный код → заполнить профиль
2. Добавлять слоты на нужные дни
3. Подтверждать/отклонять запросы клиентов прямо в чате
4. В 7:00 каждый день — автоматическое расписание на день

### Для клиента
1. Написать `/start` → «Я клиент» → выбрать тренера из списка → заполнить профиль
2. Записаться на тренировку: выбрать день → время → написать заметку
3. Тренер подтверждает → клиент получает уведомление
4. В 7:00 — напоминание о тренировке сегодня
5. За 1 час до — ещё одно напоминание

### Схема подтверждения записи
```
Клиент выбирает время
        ↓
Тренер получает запрос с кнопками [✅ Подтвердить] [❌ Отклонить]
        ↓
Клиент получает уведомление о решении
```

## 📁 Структура проекта

```
fitness_bot/
├── main.py          — точка входа
├── database.py      — SQLite (все запросы к БД)
├── keyboards.py     — все клавиатуры
├── states.py        — FSM-состояния
├── scheduler.py     — утренние уведомления (APScheduler)
├── handlers/
│   ├── trainer.py   — обработчики тренера
│   └── client.py    — обработчики клиента
├── .env             — настройки
└── requirements.txt
```

База данных `fitness_bot.db` создаётся автоматически.

## 🖥️ Деплой (чтобы бот работал 24/7)

### Вариант A — VPS (самый надёжный)
```bash
# Установить screen
apt install screen

# Запустить бота в фоне
screen -S fitbot
python main.py
# Ctrl+A затем D — отключиться (бот продолжает работать)

# Вернуться к боту
screen -r fitbot
```

### Вариант B — Railway.app (бесплатно)
1. Зарегистрируйтесь на [railway.app](https://railway.app)
2. Создайте новый проект → Deploy from GitHub
3. Добавьте переменные окружения (BOT_TOKEN, TRAINER_SECRET, TIMEZONE)
4. Railway запустит бота автоматически

### Вариант C — systemd (Linux-сервер)
```bash
# Создайте файл /etc/systemd/system/fitbot.service
[Unit]
Description=Fitness Telegram Bot
After=network.target

[Service]
WorkingDirectory=/path/to/fitness_bot
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

# Запустить
systemctl enable fitbot
systemctl start fitbot
```

## ❓ Частые вопросы

**Бот не запускается — ошибка BOT_TOKEN**
→ Проверьте файл `.env`, токен должен быть без кавычек

**Клиент не видит тренеров в списке**
→ Тренер должен сначала зарегистрироваться через /start

**Уведомления не приходят в 7:00**
→ Проверьте TIMEZONE в `.env`, должен совпадать с вашим часовым поясом
